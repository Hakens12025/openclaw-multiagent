# 硬路径与软路径

> 系统最核心的分界线：LLM 负责内容生成（软路径），代码负责流程控制（硬路径）。

## 是什么

| 维度 | 硬路径（代码驱动） | 软路径（LLM 驱动） |
|------|-------------------|-------------------|
| 用途 | 通信协议、状态管理、安全拦截、路由、进度追踪 | 内容生成、任务理解、自然语言回复 |
| 实现 | Plugin hooks、Router、硬编码逻辑 | SOUL.md、HEARTBEAT.md 指令 |
| 可靠性 | 确定性，100% | 概率性，受模型能力限制 |

## 为什么存在

LLM 的输出是概率性的，不能用来保证系统行为。安全检查、路由分发、状态管理必须由代码保障。SOUL.md 是行为引导，不是系统保证。

这条原则的强化版本是 [Agent-系统交互最小化](agent-system-minimal-interaction.md)：agent 产出内容，系统产出结构，系统从可观察信号反应而非依赖 agent 自报。

## 和谁交互

- [三层通讯协议](three-layer-protocol.md) — 协议路由属于硬路径
- [传送带原则](conveyor-belt.md) — 传送带是硬路径的 transport 原语
- [防御纵深](defense-in-depth.md) — 安全机制全部在硬路径
- [兜底机制](fallback.md) — 软路径不可靠时硬路径兜底
