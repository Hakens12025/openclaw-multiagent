# 研究方法论 Skill

当你收到研究任务时，严格遵循以下方法论执行研究，确保产出有深度、有来源、可验证。

---

## 第一步：多轮搜索策略

按三轮递进式搜索，不要一次搜索就下结论：

### 第 1 轮：广泛扫描

- 用 2-3 个宽泛关键词搜索，了解主题全貌
- 记录发现的关键方向、主流观点、争议点
- 目标：建立搜索空间，识别值得深入的子领域

### 第 2 轮：定向深入

- 针对第 1 轮发现的关键方向，用精确关键词深挖
- 每个关键方向至少搜索 2 次，使用不同措辞
- 读取搜索结果中的高价值页面（web_fetch），提取具体数据、代码、案例
- 目标：获取可引用的具体证据

### 第 3 轮：时效验证

- 对关键结论加上时间限定词搜索（"2025"、"2026"、"latest"）
- 确认发现是否仍然有效，是否有更新的替代方案
- 目标：避免引用过时信息

---

## 第二步：来源引用

每个结论必须标注来源，格式：

```
发现内容 [来源: URL 或文档路径]
```

来源分级：

- **强来源**：官方文档、论文、权威技术博客、GitHub 仓库
- **中来源**：社区讨论（Stack Overflow、GitHub Issues）、个人博客
- **弱来源**：未经验证的二手转述、仅基于 LLM 内部知识

产出中必须标明来源级别。仅基于 LLM 知识的结论标注为 `[来源: LLM 内部知识，未经外部验证]`。

---

## 第三步：交叉验证

关键发现（影响最终结论的核心事实）必须满足：

- 至少 2 个独立来源佐证
- 若只有单一来源，在结论中标注 `[单源，待验证]`
- 发现来源间矛盾时，列出两方观点和各自证据，不要只取其一

---

## 第四步：降级策略

当 `web_search` / `web_fetch` 不可用时：

1. 不要停机或报失败 -- 外部工具是增强能力，不是前提
2. 基于 LLM 内部知识和 contract 提供的本地上下文继续研究
3. 在产出中明确标注：

```
## 研究限制

本次研究未能使用外部搜索工具（原因：[具体原因]）。
以下结论基于模型内部知识，未经实时外部验证。
建议后续补充外部验证的方向：[列出具体搜索建议]
```

4. 降级模式下仍然执行完整产出结构，不缩水

---

## 第五步：结构化产出

研究主结果（写入 contract.output）必须包含以下结构：

```markdown
# 研究报告：[主题]

## 核心发现

1. [发现 1]（置信度：高/中/低）[来源: ...]
2. [发现 2]（置信度：高/中/低）[来源: ...]
3. ...

## 来源列表

| # | 来源 | 类型 | 可信度 |
|---|------|------|--------|
| 1 | URL 或路径 | 强/中/弱 | 高/中/低 |
| 2 | ... | ... | ... |

## 研究限制

- [列出本次研究的局限性，如工具不可用、时间约束、信息缺口]

## 下一步建议

- [基于当前发现，建议后续可深入的方向]
- [需要补充验证的点]
```

## 质量底线

- 不接受零来源的"概述式"产出
- 即使全部降级到 LLM 知识，也必须明确标注每条结论的置信度
- 搜索不是走形式 -- 每轮搜索必须真正执行并记录结果，不要跳过
