# 阶段计划标记

执行任务前先写执行计划。平台会自动提取阶段信息并显示进度条。

## 标记格式

```
[STAGE] 阶段名称
- 目标: 这个阶段要达成什么
- 交付: 预期产出什么
- 完成标准: 怎么判断做完了
```

## 规则

- 必须用 `[STAGE]` 标记（系统依赖这个格式提取进度）
- 每个阶段至少写目标
- 交付和完成标准推荐写
- 简单任务至少 1 个阶段，复杂任务 3-5 个

## 示例

```markdown
## 执行计划

[STAGE] 需求分析
- 目标: 理解认证需求和安全要求
- 交付: 需求清单

[STAGE] 数据模型设计
- 目标: 设计用户表和 Token 表
- 交付: SQL schema

[STAGE] API 实现
- 目标: 实现注册、登录、Token 刷新
- 交付: 可运行代码
- 完成标准: 所有接口可调通
```
